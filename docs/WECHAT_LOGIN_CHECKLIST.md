# 微信真实登录联调清单

这份清单的目标只有一个：把这条链路完整跑通并确认结果可见。

`微信登录 -> 获取用户 -> 作答评分 -> 保存训练记录 -> 成长页看到记录`

## 1. 启动前确认

- 后端 `.env` 已配置真实 `WECHAT_APP_ID`
- 后端 `.env` 已配置真实 `WECHAT_APP_SECRET`
- 后端 `.env` 已配置 `WECHAT_LOGIN_MODE=live`
- 修改后已重启后端

建议先启动：

```powershell
npm run dev:server
npm run dev:client
```

## 2. 开发者工具配置

- 在微信开发者工具里打开 `C:\Users\qq337\Documents\codex\apps\client\dist`
- 不要再用 `touristappid` 做真实登录联调，改成你的小程序真实 AppID
- 如果只是开发者工具本机联调，前端后端地址可先用 `http://localhost:4000`
- 如果是真机预览，前端后端地址必须改成电脑局域网地址，例如 `http://192.168.1.10:4000`

## 3. 先看健康状态

进入“我的 / 成长页”后，先点“检测连接”。

成功信号：

- 页面提示后端已连接
- 状态里能看到 `微信登录=live`

如果这里不是 `live`，先不要继续点登录，先回头检查后端环境变量和重启是否生效。

## 4. 做一次真实登录

在“我的 / 成长页”点击“微信登录 / 开发者工具登录”。

成功信号：

- 前端弹出“微信登录成功”
- 页面顶部从“未登录”切到“已连接账号”
- 页面能拉到当前用户资料

后端成功信号：

- 服务端终端出现 `[auth/login] attempt`
- 随后出现 `[auth/login] success`

如果失败：

- 前端会直接弹出失败原因
- 后端终端会输出 `[auth/login] failed`

## 5. 验证记录保存

登录成功后，继续跑一条完整业务链：

1. 进入任意场景
2. 输入回答
3. 完成评分
4. 保存训练记录
5. 回到“我的 / 成长页”

成功信号：

- 训练次数加 1
- 最近训练列表出现刚刚那条记录
- 平均分随记录变化

## 6. 常见卡点

- `微信登录未返回有效 code`
  重新在开发者工具里触发一次登录

- `invalid appsecret` 或类似微信换码报错
  检查 `WECHAT_APP_ID` 和 `WECHAT_APP_SECRET` 是否匹配同一个小程序

- 页面提示后端不可达
  检查后端是否启动、端口是否是 `4000`、前端保存的 API Base 是否正确

- 真机上能打开页面但登录/请求失败
  不要用 `localhost`，改成电脑局域网 IP

- 登录成功但训练记录保存失败
  优先检查 `/api/records` 是否返回 `401 unauthorized`，这通常说明 token 没带上或登录态被清掉了

## 7. 联调完成的通过标准

满足下面 4 条，就可以认为“真实微信登录联调”完成：

- 健康检查显示 `微信登录=live`
- 微信登录成功，且不是 guest fallback
- 能完成一次评分并成功保存训练记录
- 成长页能看到新记录与统计变化
