# 提审与上线配置清单

更新时间：2026-03-18

这份清单基于当前项目实际状态整理，目标是把“可联调 MVP”推进到“可提审、可上线的微信小程序版本”。

## 当前已完成

- 微信真实登录联调已通过
- 真机可连接后端，局域网联调可用
- 文本作答 -> AI 评分 -> 保存记录 -> 成长页查看 已打通
- 隐私政策页与用户协议页已补齐基础草案
- 成长页记录展示已改为中文可读格式

## P0：提审前必须完成

### 1. 线上后端与 HTTPS

- 部署一套公网可访问的后端服务
- 准备正式域名，例如 `api.yourdomain.com`
- 为正式域名配置 HTTPS 证书
- 把前端默认后端地址从本地联调地址切到正式域名
- 重新验证以下链路：
  - 微信登录
  - 拉取场景/金句
  - AI 评分
  - 保存训练记录
  - 成长页查看记录

### 2. 微信公众平台配置

- 在微信公众平台配置合法 `request` 域名
- 如果语音上传、文件下载、WebSocket、web-view 有用到，也同步配置对应域名
- 确认提审包里不再依赖 `localhost` 或局域网 IP
- 关闭仅开发期使用的“忽略合法域名校验”类设置

### 3. 主体与法务信息补齐

- 把隐私政策页里的主体名称补成真实公司/个人主体
- 补齐联系邮箱或客服联系方式
- 补齐数据删除/申诉联系渠道
- 根据你的上线主体补齐备案、许可证或平台要求的展示信息
- 检查用户协议与隐私政策中的产品名称是否统一

### 4. 密钥与环境变量治理

- 轮换已经在聊天或调试中暴露过的密钥
- 把 `WECHAT_APP_SECRET`、`DEEPSEEK_API_KEY`、`OPENAI_API_KEY` 等全部改成环境变量管理
- 确认仓库里没有把真实密钥写进可提交文件
- 区分开发、测试、正式环境变量

## P1：提审时建议一起准备

### 1. 提审素材

- 小程序名称、简介、图标、类目
- 审核员体验路径说明
- 如审核需要，准备一套稳定可复现的体验账号或演示路径
- 准备 1 到 2 段能体现核心价值的演示文案

### 2. 产品页细节

- 首页文案再收一轮，减少“开发中”语气
- 成长页的空状态、错误状态、保存状态再统一一下
- 检查协议页入口是否明显、可从首页和成长页访问
- 检查所有英文开发字段是否已替换成用户能理解的中文

### 3. 调试痕迹清理

- 确认页面上没有调试浮层、调试按钮、临时日志展示
- 确认控制台日志不会把敏感信息输出到前端
- 检查开发者工具里临时开的 mock/调试开关是否已经关闭

## P2：上线后尽快优化

### 1. 语音链路

- 接入真实语音转写 provider
- 完成“语音作答 -> 自动转写 -> 评分 -> 保存记录”完整闭环

### 2. 内容质量

- 继续打磨评分 prompt，减少“像裁判，不像教练”的感觉
- 扩充高质量场景题库
- 增加更细的成长分析

### 3. 数据与稳定性

- 评估是否从 SQLite 升级到更适合正式环境的数据库
- 增加异常日志、请求监控、数据库备份
- 为评分与登录接口增加更稳定的失败兜底策略

## 建议执行顺序

1. 先部署正式后端和 HTTPS
2. 再配置微信合法域名
3. 然后补齐协议页中的真实主体信息
4. 接着用正式域名做一次真机全链路回归
5. 最后再提交审核

## 你现在最该做的一步

先把“正式后端域名 + HTTPS + 微信合法域名”这组基础设施落下来。

因为你现在产品主链路已经能跑了，真正阻塞提审的，不再是功能，而是上线环境配置。
